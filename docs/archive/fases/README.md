# Archivo de Fases

Aquí se conservan los originales de las fases 1-10 y sus transcripciones extraídas. Úsalos para reconstruir intención histórica, no como primera parada.

## Resumen rápido por bloques

- Fases 1-2: análisis de documentación y arquitectura global.
  Hoy se aterrizan mejor en `docs/adr/` y `docs/overview/current-state.md`.
- Fases 3-6: backend, frontend, base de datos y despliegue.
  Hoy su aterrizaje práctico está repartido entre `docs/reference/`, `docs/runbooks/` y el propio código.
- Fases 7-10: mejoras propuestas, testing, documentación extensa y escalabilidad futura.
  Hoy sirven sobre todo como backlog largo y contexto estratégico.

## Estructura

- `source/`: `.docx` originales preservados.
- `transcripts/`: textos extraídos para consulta rápida o búsqueda.

## Cuándo entrar aquí

- Cuando una decisión actual no se entiende sin el razonamiento largo original.
- Cuando necesites rastrear intención de producto o arquitectura previa a la consolidación en `docs/`.
