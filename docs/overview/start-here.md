# Start Here

Usa esta guía si llevas tiempo sin tocar Viru y quieres recuperar el hilo sin leer cien documentos seguidos.

## Recorrido recomendado

1. Lee [current-state.md](current-state.md) para recordar qué stack y qué módulos están activos.
2. Abre [../adr/ADR-001-monolito-modular.md](../adr/ADR-001-monolito-modular.md), [ADR-002](../adr/ADR-002-stack-base.md) y [ADR-003](../adr/ADR-003-provider-adapter.md) para refrescar las decisiones base.
3. Si vas a trabajar con Codex o a reinterpretar reglas operativas, lee [../reference/codex-operating-contract.md](../reference/codex-operating-contract.md).
4. Si vas a tocar frontend o UX, revisa [../ui/UI_SYSTEM_V1.md](../ui/UI_SYSTEM_V1.md), [../ui/UI_CONTRACT_V1.md](../ui/UI_CONTRACT_V1.md) y [../specs/README.md](../specs/README.md).
5. Si vas a tocar búsqueda rápida o backend, revisa [../reference/README.md](../reference/README.md) y en especial la carpeta `docs/reference/backend/`.
6. Si vas a desplegar, depurar o validar cambios, entra en [../runbooks/](../runbooks/) y [../qa/README.md](../qa/README.md).
